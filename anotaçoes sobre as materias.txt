Sobre Power BI.  
. o que é ?

	O Power BI é uma coleção de serviços de software,aplicativos e conectores que trabalham juntos para transformar suas fontes de dados não relacionadas em informações
coerentes, visualmente envolventes e interativas. Os dados podem estar em uma planilha do Excel ou em uma coleção de data warehouses híbridos locais ou baseados
na nuvem. Com o Power BI, você pode se conectar facilmente a fontes de dados,visualizar e descobrir conteúdo importante e compartilhá-lo com todas as pessoas
que quiser.

O Power BI consiste em vários elementos que trabalham juntos, começando com estes três elementos básicos:

 . Um aplicativo de desktop do Windows chamado Power BI Desktop.
 . Um serviço SaaS (software como serviço) online chamado de serviço do Power BI.
 . Aplicativos móveis do Power BI para dispositivos Windows, iOS e Android.

Git E GitHub
 
. Repositório: 
O repositório é onde ficam todos os arquivos de um projeto. Geralmente, fica armazenado em um servidor remoto para que, através de um software adequado (GIT),
seja acessado e controlado, permitindo que os programadores tenham acesso adequado às funcionalidades.

O GitHub, além de ser um servidor para projetos, serve como uma rede social para programadores. O seu uso é bastante recomendado. Nele é possível compartilhar
projetos, criar repositórios (públicos ou privados), e participar do desenvolvimento de outros usuários.

. Primeiros comandos com GIT:

Comandos Basícos :

Comando 	 Funcionalidade
 git init		 Cria um novo repositório

 git clone caminho_projeto		 Copia um projeto de um servidor remoto

 git status		 Exibe o status dos arquivos modificados

 git log	 	Log do que foi feito

 git branch nome_branch	 Cria uma nova branch

 git branch -b nome_branch	 Copia os dados e cria uma branch nova

 git add nome_arquivo	 Permite que um arquivo seja monitorado e o adiciona para a área de envio

 git add .	 	Adiciona todos os arquivos de uma vez só

 git commit -m “”	 Permite confirmar a adição do arquivo e inserir um comentário

 git push origin nome_branch		 Envia os arquivos para o servidor remoto

 git pull origin nome_branch		 Atualiza o repositório local com os dados do servidor remoto

 git merge nome_branch	 	Mescla arquivos de branchs distintas

Não é preciso se preocupar em decorar todos esses comandos. Alguns são utilizados apenas em situações
bem específicas e, com a prática do exercício contínuo de envio e manutenção dos arquivos em um projeto,
conseguiremos lembrar deles naturalmente, fique tranquilo(a)! =) 

O github é um serviço para hospedar e permitir o gerenciamento correto de repositórios. Utilizando a dupla
git & github podemos enviar nossos projetos e disponibilizá-los para outros desenvolvedores, permitindo o 
trabalho em equipe e o gerenciamento de todas as versões.
 
No site oficial do git existem algumas recomendações de ferramentas que podem ser utilizadas para facilitar
a visualização do trabalho que está sendo feito.

Github Desktop:
Precisa ser instalado e possui integração com o github. É muito popular pela interface adotada e funcionalidades disponíveis.

Gitkraken:
Possui uma interface intuitiva e de fácil compreensão pelos usuários. Ajuda em todo o processo de monitoramento do projeto.

Gitk :
Dentro de um repositório local é possível executar o comando gitk e ter acesso a uma interface que facilita a visualização
da linha de tempo criada.

Comandos Úteis 
. GIT ADD
  Faz o staging de tudo , exceto arquivos apagados ( continuarão existindo no remoto ). 

. GIT COMMIT -M " mensagem de commit" 
 Cada sequência de stagings pode receber uma mensagem de commit informando resumidamente quais modificações estão sendo submetidas.

. GIT PUSH -U ORIGIN NOME_DA_BRANCH
 Submete todas as modificações em stagin para a branch de destino na origem ( repositório remoto )

. GIT PULL
 Atualiza o repositório local com modificações que foram realizadas no repositório remoto desde a última sincronização . Vai mesclar (merge) o 
conteúdo da origem no seu repositório e pode causar conflitos se você nao submeteu modificações recentes .
